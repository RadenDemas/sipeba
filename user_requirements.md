2. User Requirements

User requirements menjelaskan kebutuhan fungsional dari sudut pandang pengguna.

2.1 Kebutuhan Umum Pengguna

Pengguna harus dapat login dan logout sesuai role masing-masing.

Sistem harus menyediakan antarmuka web yang responsif (desktop & mobile).

Pengguna hanya dapat mengakses fitur sesuai dengan hak aksesnya.

2.2 Kebutuhan Karyawan

Dapat melihat dan mencari daftar barang.

Dapat mengajukan peminjaman barang dengan durasi dan alasan.

Dapat melihat status peminjaman secara real-time.

Dapat menerima notifikasi email pengingat pengembalian.

Dapat melihat riwayat peminjaman pribadi.

Dapat mengelola profil akun.

2.3 Kebutuhan Petugas Gudang

Dapat melihat daftar pengajuan peminjaman.

Dapat menyetujui atau menolak peminjaman.

Dapat mencatat proses pengambilan dan pengembalian barang.

Dapat melihat riwayat peminjaman per barang.

Dapat melihat histori aktivitas kerja.

2.4 Kebutuhan Admin

Dapat mengelola data user dan role.

Dapat mengelola data barang.

Dapat melihat laporan dan histori peminjaman secara menyeluruh.

Dapat memonitor aktivitas seluruh pengguna.
