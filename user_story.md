1. User Stories

User stories disusun berdasarkan role utama: Karyawan, Petugas Gudang, dan Admin (Super Admin).

1.1 User Stories – Karyawan

Sebagai karyawan, saya ingin login ke sistem, agar dapat mengakses fitur peminjaman barang.

Sebagai karyawan, saya ingin melihat daftar barang yang tersedia, agar saya tahu barang apa saja yang bisa dipinjam.

Sebagai karyawan, saya ingin mencari dan memfilter barang, agar lebih mudah menemukan barang yang dibutuhkan.

Sebagai karyawan, saya ingin mengajukan peminjaman barang dengan menentukan durasi dan alasan, agar proses peminjaman terdokumentasi dengan jelas.

Sebagai karyawan, saya ingin melihat status peminjaman (pending/disetujui/ditolak), agar mengetahui progres permohonan saya.

Sebagai karyawan, saya ingin mengubah atau membatalkan pengajuan sebelum disetujui, agar dapat memperbaiki kesalahan input.

Sebagai karyawan, saya ingin mendapatkan notifikasi email pengingat pengembalian, agar tidak terlambat mengembalikan barang.

Sebagai karyawan, saya ingin melihat riwayat peminjaman saya, agar dapat memantau aktivitas peminjaman.

Sebagai karyawan, saya ingin mengelola profil (ganti password/email), agar akun saya tetap aman.

1.2 User Stories – Petugas Gudang

Sebagai petugas gudang, saya ingin login sesuai role, agar dapat mengelola peminjaman barang.

Sebagai petugas gudang, saya ingin melihat daftar pengajuan peminjaman, agar dapat memproses permintaan.

Sebagai petugas gudang, saya ingin menyetujui atau menolak peminjaman, agar stok barang terkontrol.

Sebagai petugas gudang, saya ingin melihat detail barang dan peminjam, agar dapat mengambil keputusan yang tepat.

Sebagai petugas gudang, saya ingin memverifikasi pengambilan dan pengembalian barang, agar status barang selalu akurat.

Sebagai petugas gudang, saya ingin melihat riwayat peminjaman per barang, agar dapat melakukan monitoring.

Sebagai petugas gudang, saya ingin melihat histori aktivitas saya, agar aktivitas kerja dapat ditelusuri.

1.3 User Stories – Admin (Super Admin)

Sebagai admin, saya ingin login ke sistem dengan hak akses penuh, agar dapat mengelola seluruh data.

Sebagai admin, saya ingin mengelola data pengguna (CRUD user), agar akun pengguna terkontrol.

Sebagai admin, saya ingin mengelola data barang, agar data inventaris selalu terbaru.

Sebagai admin, saya ingin melihat laporan dan riwayat peminjaman, agar dapat melakukan monitoring dan evaluasi.

Sebagai admin, saya ingin melihat tracking aktivitas pengguna, agar audit sistem dapat dilakukan.

Sebagai admin, saya ingin mengelola role dan hak akses, agar keamanan sistem terjaga.
